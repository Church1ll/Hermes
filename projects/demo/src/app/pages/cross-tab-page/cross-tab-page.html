Feature 05

Cross-Tab WebSocket Communication

Open this page in multiple browser tabs. Only one tab should keep the real WebSocket connection, while the other tabs receive mirrored data through cross-tab communication.

How to test

Open multiple tabs

Watch the browser Network panel. The leader tab owns the socket, while follower tabs receive the same messages without creating their own long-lived connection.

Leader failover

Close the leader tab

After the heartbeat timeout, another tab becomes leader and starts managing the external source.
